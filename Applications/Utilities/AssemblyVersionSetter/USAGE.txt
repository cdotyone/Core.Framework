
AssemblyVersionSetter: Searches a project structure for an AssemblyFileVersion 
                       and then sets it to a different version.
Author: Chris Doty <cdoty@civicinc.com>

AssemblyVersionSetter.exe [options] version [path]

Options:
	
	-e - Causes numbers to be added to the end of an existing version # 
	     This is the default action

		Example 1:
		command: AssemblyVersionSetter.exe 1.0 

		[assembly: AssemblyFileVersion("1.0.0.0")] is current version

		it would be set to [assembly: AssemblyFileVersion("1.0.1.0")]

		Example 2:
		command: AssemblyVersionSetter.exe 1.0.3.4

		[assembly: AssemblyFileVersion("1.0.0.0")] is current version

		it would be set to [assembly: AssemblyFileVersion("1.0.3.4")]


	-s - causes numbers to be added to the begining of an existing version #

		Example 1:
		command: AssemblyVersionSetter.exe -s 2.0 

		[assembly: AssemblyFileVersion("1.0.0.1")] is current version

		it would be set to [assembly: AssemblyFileVersion("2.0.0.1")]

		Example 2:
		command: AssemblyVersionSetter.exe -s 1.0.3.4

		[assembly: AssemblyFileVersion("1.0.0.1")] is current version

		it would be set to [assembly: AssemblyFileVersion("1.0.3.4")]
	
	-t - outputs more tracing information
Version:
	The version must be a normal version sequence.  
	At minimum it must contain 2 numbers and no more than 4

	Exceptable examples:

	AssemblyVersionSetter.exe 1.0
	AssemblyVersionSetter.exe 1.0.1
	AssemblyVersionSetter.exe 1.2.1.1

Path:
	Can be the path to the directory to scan.
	By default it will search the current working directory.  
	If the current working directory is the bin or under the bin 
	directory the project root folder will be used
